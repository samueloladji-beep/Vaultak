How to Add a Kill Switch to Your AI Agent

Vaultak Engineering · 2026

Every AI agent running in production needs a kill switch. This is not optional. It is the minimum safety requirement for any agent with real-world capabilities.

Automatic Kill Switch

pip install vaultak
from vaultak import Vaultak

vt = Vaultak(api_key="vtk_your_key")

vt.policy.create({"name":"auto-kill-high-risk","condition":"risk_score >= 0.9","effect":"pause","notify":["slack","pagerduty"],"priority":1})

Manual Kill Switch via API

vt.agent.pause(agent_id="production-agent")
vt.agent.resume(agent_id="production-agent")

Dashboard Kill Switch

Every agent monitored by Vaultak has a pause/resume button in the dashboard at app.vaultak.com. Any team member with dashboard access can stop the agent instantly.

Building Your Own Kill Switch

import threading

kill_event = threading.Event()

def agent_runner():
    with vt.monitor("agent"):
        while not kill_event.is_set():
            agent.step()

def kill_agent():
    kill_event.set()
    vt.agent.pause("agent")

Frequently Asked Questions

How do I add a kill switch to my AI agent?
Use vt.policy.create() with auto-pause conditions, or call vt.agent.pause() from anywhere in your code or via API.
Can I stop an AI agent from the dashboard?
Yes. Every Vaultak-monitored agent has a pause/resume button in the dashboard at app.vaultak.com.
How do I automatically stop an AI agent doing something dangerous?
Set a risk score threshold in your Vaultak policy with effect pause. The agent stops automatically when exceeded.
What happens to agent state when I pause it?
Vaultak preserves the agent audit log. You can resume from where it stopped or roll back to a previous snapshot.
