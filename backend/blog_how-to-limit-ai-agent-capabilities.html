How to Limit What an AI Agent Can Do

Vaultak Engineering · 2026

The most effective AI agent security control is also the simplest: give the agent fewer capabilities. An agent that cannot delete files cannot accidentally delete your database.

Tool-Level Restrictions with Vaultak

pip install vaultak
from vaultak import Vaultak

vt = Vaultak(api_key="vtk_your_key")

# Default deny
vt.policy.create({"name":"default-deny","action":"*","resource":"*","effect":"deny","priority":100})

# Explicitly allow only what is needed
vt.policy.create({"name":"allow-search","action":"web_search","resource":"*","effect":"allow","priority":1})
vt.policy.create({"name":"allow-read-docs","action":"read_file","resource":"documents/*","effect":"allow","priority":1})

Resource-Level Restrictions

vt.policy.create({"name":"read-only-db","action":"write","resource":"*_database","effect":"deny","priority":1})
vt.policy.create({"name":"own-tenant-only","action":"*","resource":"tenant_{tenant_id}/*","effect":"allow","priority":1})

Time Window Restrictions

vt.policy.create({"name":"business-hours-only","action":"*","resource":"*","effect":"deny","time_window":{"outside_hours":"09:00-17:00","timezone":"UTC"}})

Frequently Asked Questions

How do I restrict what tools an AI agent can use?
Use Vaultak allowlist approach - deny all actions by default, then explicitly allow only the tools the agent needs.
Can I limit an AI agent to read-only access?
Yes. Create Vaultak policies that deny all write, delete, and update actions while allowing read operations.
How do I restrict an AI agent to specific data?
Use resource patterns in Vaultak policies to limit the agent to specific tables, directories, or data namespaces.
Can I schedule when an AI agent is allowed to operate?
Yes. Vaultak time-window policies restrict agent operation to specific hours, days, or date ranges.
